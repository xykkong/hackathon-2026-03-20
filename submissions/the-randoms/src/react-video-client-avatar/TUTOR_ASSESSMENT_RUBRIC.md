# Tutor Assessment Rubric (Post-Call Report)

Source implementation:
- `react-video-client-avatar/components/VideoAvatarClient.tsx`
- Function: `buildTutorReport(...)`

This app currently does not use an LLM "assessment prompt" for the report.
The report is generated by deterministic scoring logic in code.

## Inputs
- Conversation transcript (`messageList` + in-progress message)
- Speaker split (user vs agent)
- Optional Thymia snapshot:
  - `biomarkers`
  - `wellness`
  - `clinical`
  - `progress`
  - `safety`

## Core Metrics
1. Engagement (0-10, forced to 0 if user turns = 0)
- Based on user turn count and user question count.

2. Clarity (0-10, forced to 0 if user turns = 0)
- Based on average user message length and number of overly long messages.

3. Conversation Flow (0-10, forced to 0 if user turns = 0)
- Based on user/agent turn balance and total turn count.

4. Speaking Confidence (always included)
- If user turns = 0 -> score 0.
- If Thymia confidence exists -> uses Thymia confidence.
- Otherwise -> estimated from engagement/clarity/flow.

## Thymia Integration
When available, report includes:
- confidence %
- stress %
- fatigue %
- distress %
- safety alert
- analyzed speech seconds

These signals influence:
- confidence metric rationale
- improvement suggestions (e.g., stress pacing, confidence scaffolding)
- evidence section

## Generated Sections
- Overview
- Assessment metrics
- What Went Well
- What Can Be Improved
- Next Session Goals
- Evidence from Conversation
- Thymia Speaking Signals (when available)
